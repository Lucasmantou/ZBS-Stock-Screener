---

## 一、设计理念

### 核心原则
1. **信息密度优先** - 关键数据前置，结论置顶，数据驱动叙事
2. **红涨绿跌** - K线颜色严格遵循：阳线(close≥open)红色，阴线(close<open)绿色；成交量柱同步分色
3. **双主题兼容** - 深色模式适合长时间盯盘，浅色模式适合打印或明亮环境
4. **几何精确** - 零圆角、硬边框、等宽字体数据展示，传达专业与精确
5. **层级分明** - 通过边框、字号梯度和间距构建清晰的阅读层次

## 五、组件规范

### 5.1 顶部导航

```css
.top-bar {
  position: sticky; top: 0; z-index: 100;
  background: rgba(9,12,20,0.88);
  backdrop-filter: blur(20px) saturate(1.4);
  border-bottom: 1px solid var(--border);
  height: 52px;
}
```

- Logo：`ZBS` monospace 字重900，蓝色
- 股票名称 + 代码
- 导航链接：hover时蓝色边框，active时蓝色填充白色文字

### 5.2 Hero行情卡片

- 零圆角、无渐变背景
- 价格68px等宽数字
- 涨跌幅：半透明边框标签
- 4格KPI网格，1px边框分隔

### 5.3 结论置顶卡片

- 左侧4px蓝色竖线（非顶部3px）
- 标签：蓝色等宽字体大写
- 药丸标签：蓝色边框 + 蓝色半透明背景

### 5.4 区块标题

```html
<div class="section-head">
  <span class="section-num">01</span>
  <h2>区块标题</h2>
  <span class="sub">副标题</span>
</div>
```

- 编号：28x28px蓝色半透明背景 + 蓝色边框，等宽字体
- 标题：17px字重700

### 5.5 表格

```css
.tb th {
  font-size: 12px; font-weight: 700;
  text-transform: uppercase; letter-spacing: 0.8px;
  border-bottom: 2px solid var(--border);

```

### 5.6 评分条

- 标签：110px固定宽度，右对齐
- 轨道：6px高度，灰色背景
- 填充：蓝色渐变，宽度百分比
- 分数：等宽字体加粗

### 5.7 情景卡片

- 顶部3px色条：基准蓝色、悲观绿色、乐观红色
- 内部居中对齐

### 5.8 评分环

- 88x88px，蓝色轨道
- 数字：26px等宽字重900

## 七、响应式断点

| 断点 | 规则 |
|------|------|
| ≤900px | 双列/三列网格单列；Hero价格48px；隐藏桌面导航，显示汉堡菜单 |
| ≤600px | Hero价格38px；移动端表格卡片堆叠 |

## 八、交互规范

- 滚动入场动画（translateY + opacity）
- 数字动画（easeOutCubic缓动）
- ResizeObserver图表自适应
- 导航点击立即高亮（不等scroll事件）
- 移动端表格data-label自动填充
- 卡片折叠（section collapse）
- 返回顶部按钮
- 深色/浅色主题切换，图表同步重绘
